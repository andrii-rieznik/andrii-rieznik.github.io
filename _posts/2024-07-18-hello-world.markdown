It’s been a long time since I first thought about making a personal blog. Maybe it’s been ten years ago or even more. The reason for my wish was that it just seemed cool to me. I imagined how I would add this desirable link to my CV or my social network profile. When I was studying at the university my group mate had one. Back then sometimes I visited his blog to see how it was going. I was not even interested in reading his posts. Just in how it had been done. Later when I started to learn technologies deeply a significant amount of useful information was shared by other people on their own blogs or publishing platforms. I found it not only a useful way to share knowledge in the digital era but also practical in personality and identity development.

Many times I had thought about what topic the first post should be. But honestly, even until now, I’m still unsure. Maybe I have to write five or six posts for a brand-new blog and post them simultaneously but with different publication dates. It might give a sense of stability and presence for a certain period. But frankly, who cares how many posts you have already made? You would rather keep some drafts for the moment when you are out of ideas or materials for further posts, as I did :)


### Say hello

I decided to write the first post in a way like I usually write a program in a new language. Ironically, writing the first program is much easier than writing a post. Let's write one for clarity. And, yeah, you could write in assembly languages or machine code but it may be challenging even to write such kind of program and would require deeper technical skills. Nowadays, there is a wide range of high-level programming languages whose syntax is simple, and writing sometimes looks like writing in English. The Ruby language is one of them, elegant and powerful. It may be handy in daily tasks, DevOps, and automation. Plus, there are a lot of useful utils written in Ruby and, of course, one of the best web frameworks ever is also written in Ruby.

0. Make sure you have [installed][ruby-installation] Ruby on the machine if you haven't.

1. Write the code in the text editor, like VS Code, Notepad++, etc, and save the file with `.rb` extension:

    ```ruby
    puts "Hello, World!"
    ```

2. To execute a program run the following command, it will print the output to STDOUT:

    ```console
    > ruby hello.rb
    Hello, World!
    ```

Finally, I've written the first post for my blog and later today I’m going to deploy it.
I chose [Jekyll][jekyll-gh] among other static site generators since it has everything I need.
It can be customized by plugins (gems) and works natively with GitHub pages.
